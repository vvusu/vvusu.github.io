PVE 安装后需要做的事情

系统更新

取消订阅

nano /etc/apt/sources.list

取消订阅
nano /etc/apt/sources.list.d/pve-enterprise.list

注释掉订阅

apt-get update

初始化磁盘

fdisk /dev/sdc

进入fdisk
输入 p 查看磁盘分区
输入 d 删除分区
输入 w 保存
在命令行中查看磁盘 S.M.A.R.T 信息： smartctl -a /dev/sdc

打开 iommu & pci passthrough

直通硬件，主板支持 iommu

Inter cpu填写 intel_iommu=on

AMD CPU填写 amd_iommu=on

填写完 使用 update-grub 更新 grub

命令行运行 dmesg | grep -e DMAR -e IOMMU 如果没有输出，则说明有问题。

更新完后 reboot 系统。

配置PVE IPV6

按下面的操作修改配置文件后即可正常获取 IPv6。  
注意：本教程是让你的 PVE 主机 和 LXC 容器获取 IPv6 地址，如果你虚拟机和其他下连设备出现问题，请先确定是不是你自己的问题

  •   修改配置文件 nano /etc/sysctl.conf
  •   把文件内容替换为 / 在末尾加上下面的内容

另外备注一下：LXC 容器 获取 IPv6 地址要设置成 SLAAC 才能获取地址 (设置为 SLAAC 后既可以 DHCPv6 有状态，也可以 SLAAC 无状态，设置为 DHCP 只能拿到 Link-Local 链路地址)  
LXC 容器修改完网络设置重启对应容器即可，重启之后检查一下 PVE 主机和 LXC 容器有没有拿到地址

可以看到同时通过 DHCPv6 和 SLAAC 拿到了地址
